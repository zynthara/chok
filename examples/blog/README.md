# blog — the chok v2 quickstart

A JWT-guarded blog API — register, log in, own your posts — in about
a hundred lines of application code. Everything else (HTTP server,
SQLite + auto-migration, auth endpoints, health probes, Prometheus
metrics, OpenAPI docs) is assembled from `chok.yaml`.

## The five-minute path

Requires Go 1.26+. From the repository root:

```sh
cd examples/blog
go run .
```

You should see `starting application` and the server on `:8080`.
In a second terminal:

```sh
# 1. Liveness — the app is up:
curl -s localhost:8080/healthz
# {"status":"up",...}

# 2. Register and log in (account module — assembled from chok.yaml):
curl -s -X POST localhost:8080/auth/register \
  -H 'Content-Type: application/json' \
  -d '{"email":"alice@example.com","password":"password123","name":"Alice"}'

TOKEN=$(curl -s -X POST localhost:8080/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"alice@example.com","password":"password123"}' | \
  python3 -c 'import json,sys; print(json.load(sys.stdin)["token"])')

# 3. Create a post (401 without the token — try it):
curl -s -X POST localhost:8080/api/v1/posts \
  -H "Authorization: Bearer $TOKEN" -H 'Content-Type: application/json' \
  -d '{"title":"Hello chok","content":"my first post"}'
# note the "id": "pst_..." — public IDs are prefixed, numeric keys never leak

# 4. List your posts (owner-scoped: you only ever see your own):
curl -s localhost:8080/api/v1/posts -H "Authorization: Bearer $TOKEN"

# 5. Publish it (optimistic locking rides the version column):
RID=pst_...   # from step 3
curl -s -X PUT localhost:8080/api/v1/posts/$RID \
  -H "Authorization: Bearer $TOKEN" -H 'Content-Type: application/json' \
  -d '{"status":"published"}'

# 6. Browse the generated API docs:
open http://localhost:8080/swagger        # spec: /swagger/doc.json
```

Stop the server with Ctrl-C — components shut down in reverse
dependency order and the process exits cleanly.

## What to look at

| File | Lesson |
|---|---|
| `chok.yaml` | The single config: sections declare modules, `enabled` toggles them at runtime, every leaf is `BLOG_*` env-overridable. |
| `chok_modules_gen.go` | Generated by `chok sync` from the yaml — add a section (say `redis:`), rerun `chok sync`, and it links. |
| `main.go` | The Post model + the one customization pattern: `chok.Override(db.Module(db.WithTables(...)))` replaces the synced db entry with one that knows your tables. |
| `posts.go` | Routes behind `account.Authn(k)`: typed handlers (`HandleRequest`), list endpoint (`HandleList`), store field whitelists, optimistic-lock update, versioned delete. |
| `blog_test.go` | The acceptance test that walks this README's path over real HTTP on every CI run. |

Things you get without writing them: owner isolation (the `Post`
model embeds `db.OwnedSoftDeleteModel`, so every store query is
fenced to the authenticated user and writes stamp ownership), 404/409
error envelopes (`chok.WithErrorMapper(store.MapError)`), request IDs
on every log line, `/metrics`, `/readyz`, and login rate limiting.

## Where to go next

- `chok init myapp` scaffolds a fresh project with this same shape.
- Add batteries by adding yaml sections — `audit:`, `authz:`,
  `scheduler:`, `redis:` — and rerunning `chok sync`.
